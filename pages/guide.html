Learning Resources

Swirl

Swirl is an interactive tutorial that you run inside of your console in RStudio. It has lessons varying from the basics, to regressions, to exploratory data analysis. It is a great (once again, FREE!) resource that can definitely get you up to speed.

DataCamp

DataCamp is an alternative to Swirl that runs in browser and takes a signicantly shorter amount of time to complete in its entirety. This is also free, but in my opinion lacks the depth that Swirl offers. You can always try both and see what you like better!

R for Data Science

This might not be for complete beginners, but once you familiarize yourself with R, this book is absolutely my favorite resource for R. And you guessed it, it's FREE (however physical copies are available if you want to show your support to the authors)! It is written by Hadley Wickham and Garrett Grolemund, who are probably some of the most knowledgeable people about all things R. If you read this book your understanding of R will grow exponentially. It teaches you the foundations of transforming and tidying data so that it is easier to work with for modeling and visualization. This book also helps you teaches you the basics of creating statistical models for your own analyses!

R Markdown

I just wanted to mention R Markdown and it's amazing functionality. Here is a chapter showcasing what R Markdown is and it's benefits. But, here is a quick summary that should convince you:

  • You can combine your code and its output and written observations in one document, which is super nice for work flow!
  • You can take everything you wrote in this document, and knit it together to make a complete report. It typesets everything, and within seconds have a professional document in pdf, html, or other filetypes.
  • You can format most aspects of the knit document. Font size, headers, tables, lists, images, and many more aspects are controlled by what you want to show. If you want to hide all of your code, and just show the output, you can do that!

The R Online Community

This isn't so much a single resouce, but the R Online Community is insanely awesome. If you're looking to connect with other R enthusiasts, the R for Data Science Online Learning Community is full of awesome tweets about R and data science. They also have a weekly data science project for you to work on! They have a GitHub repository that contains all of the datasets they use for that weeks project. So you just have to donwload the data and basically do whatever you want with the data.

Once you finish your rendition of the project, you can submit it to twitter and get feedback on your work. I have seen some awesome data vizualizations retweeted on this page, and has taught me how to do certain tidying/visualizing concepts that I have been able to apply to my own projects. There is also a slack group where they have various channels where you can talk about all things R with extremely awesome and supportive people.

Functions to Make Life Easy

Functions for Simulation

R is really great for simulations, and there are many functions in base R that are very helpful.

Functions Belonging to Statistical Distributions

Really useful functions that appear fairly often in basic simulations belong to the normal, uniform, and binomial distribution. Every distribution family has four key functions: Density, distribution function, quantile function and random generation for each distribution. Let's break it down.

Density Functions

Density functions start with the "d" prefix (dnorm, dunif, dbinom). I typically use dbinom the most, because it returns the probablity of X successes in n bernoulli trials, where X is a random variable. For example, if we want to calculate the probability of 5 heads in 10 coin flips, we would input > dbinom(2, 4, 0.5) 0.375 which is the same as calculating this using the PDF for the binomial distribution: > choose(4,2) * (0.5)^2 * (0.5)^2 0.375

Distribution Functions

The "p" family uses the cumulative distribution functions (CDF) to calculate the probability that X will take a value less than or equal to x. I use this most often with pnorm(), which is used to return the p-value of a test statistic using the normal distribution. For example, if we conduct a statistical test and get a test statistic of 2.32, we would use this code: > 1 - pnorm(2.32) [1] 0.01017044

Other Functions

dput()

When it comes to output in R, it can sometimes be very cumbersome to work with. For example, let's use the built in iris datset. Say I want to refactor the species column in the dataframe in a certain order. I could use
iris$species <- factor(iris$species, levels = c("setosa", "versicolor", "virginica"))
or any other combination of the three. If I wanted to be quick, I could copy and paste some output and add some commas to get by. For example, calling
> levels(iris$Species)
[1] "setosa" "versicolor" "virginica"
will work, but you definitely have some extra work to do! This wouldn't take a lot of time to type out by hand, but imagine that your dataframe had a lot of different levels. It would be really annoying and slow to type out, right? The function dput() is here to save the day! It will take in a dataframe column, and returns the levels of the column separated by a comma, and inserts inside of the c() call. Beautiful!

Here's what it will look like:
> dput(levels(iris$Species))
c("setosa", "versicolor", "virginica")
Now, say I wanted it in reverse alphabetical order. It's as simple as
> dput(sort(levels(iris$Species), decreasing = TRUE))
c("virginica", "versicolor", "setosa")
So, now it is super easy to use for c() calls!
